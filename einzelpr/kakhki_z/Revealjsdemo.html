Syntaktischer Aufbau einer Arrow-Function:

  1. Ein Parameter
    • (param) => { statements }
    • param => { statements }
    • param => expression
  2. Mehrere Parameter
    • (param1, param2, …, paramN) => { statements }
    • (param1, param2, …, paramN) => expression
  3. Kein Parameter
    • () => { statements }
    • () => expression;
  4. // one statement, returning an object ([param]) => ({ key: value });
Die damit weiterhin verbundenen Änderungen:
  • 1) Anonymität
  • 2) Verkürzung der Syntax

Voraussetzung:

  • Funktion als Argument einer anderen Funktion als Voraussetzung dafür
